# Apache Deploy Checklist

1. Run `npm run build` locally.
2. Delete the previous deploy contents on the server before uploading the new build.
3. Upload the full contents of `dist/`, including `.htaccess`, `assets/`, `images/`, and root files.
4. Do not mix old `assets/` files with the new build.
5. Purge any hosting or CDN cache after upload if your provider adds one.

What this project expects:

- `index.html` is served with `Cache-Control: no-cache, no-store, must-revalidate`.
- Fingerprinted Vite files under `/assets/` are served with `Cache-Control: max-age=31536000, public, immutable`.
- Public files outside `/assets/` use versioned URLs and a shorter cache window.

Quick verification after deploy:

1. Open the site in a private window.
2. View source and confirm the script and stylesheet filenames in `/assets/` contain a hash.
3. Confirm favicon and OG image URLs include `?v=`.
4. Confirm there are no old root-level `.js` or `.css` files left on the server.
